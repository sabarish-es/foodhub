TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

If you still see "Unexpected token '<'" error:

1. Check backend is running:
   lsof -i :3001
   (Should show Node.js process)

2. Check backend console for errors:
   - Look at the output when you run "node server.js"
   - Look for any error messages

3. Restart backend completely:
   - Kill any existing Node process
   - Run "node server.js" again

4. Clear browser cache:
   - Close and reopen browser
   - Or press Ctrl+F5 (hard refresh)

5. Check .env file:
   - Make sure DB credentials are correct
   - Check NEXT_PUBLIC_API_URL is set to http://localhost:3001/api


IF STILL HAVING ISSUES
═════════════════════════════════════════════════════════════════════════════

1. Check browser console (F12 → Console tab)
   - Look for any error messages
   - Copy error and check what API call failed

2. Check backend console
   - Look for SQL errors
   - Look for authentication errors

3. Verify login:
   - Are you logged in as admin?
   - Is your token valid?
   - Try logging out and back in

4. Database check:
   - Is MySQL running?
   - Does database "restaurant_management" exist?
   - Are all tables created?

See API_ERROR_FIX.md for detailed troubleshooting.
